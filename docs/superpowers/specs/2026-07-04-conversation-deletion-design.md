# 会话删除功能设计

## 背景与目标

聊天入口已经支持新建会话、历史会话和长短期记忆，但缺少用户主动删除会话的能力。本功能为当前用户提供不可恢复的会话删除操作，同时保留企业级运行追踪、审计和指标，避免删除聊天内容破坏治理证据。

## 删除语义

删除会话时永久删除以下数据：

- 会话主记录；
- 会话中的用户与 Assistant 消息；
- 会话摘要；
- 以该会话为 `source_conversation_id` 提取的长期记忆。

以下治理数据继续保留：

- `task_runs`；
- `audit_events`；
- 运行产物；
- 成本、延迟和成功率等指标。

删除后不可恢复。保留的治理数据只用于运行追踪与审计，不再作为聊天历史或长期记忆参与后续上下文组装。

## API 与权限

新增接口：

```http
DELETE /api/conversations/<conversation_id>
```

接口继续使用 `CHAT_USE` 权限，并依次校验：

1. 会话存在；
2. 会话属于当前租户；
3. 会话属于 `general_agent` 统一聊天入口；
4. 会话属于当前有效用户。

不存在或越权访问统一返回 `404`，避免通过响应差异枚举其他用户的会话。

删除前，服务查询该会话及其父子运行。任何关联运行处于 `running` 或 `waiting_for_approval` 时返回 `409`，提示用户先结束任务或完成人工审批。前端状态只能用于提前阻止误操作，后端校验是最终约束。

删除成功返回：

```json
{
  "status": "deleted",
  "conversation_id": "..."
}
```

## 服务与存储边界

删除由统一的会话删除服务协调，不允许 Web 层分别调用多次存储删除操作。

会话存储新增原子删除能力，在单个数据库事务中按以下顺序执行：

1. 删除会话摘要；
2. 删除会话消息；
3. 删除存储在同一后端、来源为该会话的长期记忆；
4. 删除会话主记录。

SQLite 不依赖默认关闭的外键级联，必须显式删除子记录。PostgreSQL 使用相同的显式语义，保证两个后端行为一致。

向量存储协议新增“按来源会话删除记忆”的能力。默认 SQLite、PostgreSQL 和 `pgvector` 实现相同契约。外部向量存储无法与会话数据库共享事务时，由删除服务先清理记忆；清理失败则停止会话删除并返回可重试错误，优先避免已删除会话的记忆继续被召回。若记忆已清理但会话事务随后失败，会话仍保留，用户可重试删除。

运行存储新增按租户、用户和会话查询阻塞中运行的能力，并同时覆盖父运行和子 Agent 运行。

删除成功后写入不含消息正文的结构化应用日志，包含租户、用户、会话 ID 和各类删除记录数量。

## 前端交互

每个历史会话使用独立容器承载两个相邻控件：

- 打开会话的主按钮；
- 删除会话的图标按钮。

不得嵌套按钮。桌面端在会话项悬停或内部控件获得焦点时显示删除按钮；移动端保持弱化的删除按钮可见。

点击删除按钮不会切换当前会话，而是打开站内确认对话框。对话框显示会话标题并说明：

- 删除后不可恢复；
- 消息、摘要及相关长期记忆会被删除；
- 企业审计与运行追踪会继续保留。

默认焦点位于“取消”按钮；“确认删除”使用危险操作样式；支持 `Esc` 关闭、焦点约束和键盘操作。提交期间禁用操作按钮，防止重复请求。

## 删除后的界面状态

- 删除当前会话：取消当前会话的异步加载守卫，清空聊天区、待审批状态和追踪抽屉，进入尚未持久化的“新会话”状态；下一次发送消息时再创建会话。
- 删除非当前会话：只从缓存和历史列表移除该项，当前聊天内容不变。
- `404`：按“会话已经不存在”处理，刷新历史列表，并在必要时重置当前聊天。
- `409`：保留确认对话框，显示“该会话仍有任务正在执行或等待审批，请先结束任务”。
- 网络或存储错误：保留会话与确认对话框，显示可重试错误。

## 并发与一致性

删除请求期间，前端禁止重复提交。服务端在执行删除前重新校验所有权和阻塞运行状态。

会话消息写入服务继续要求会话存在且状态有效。即使其他客户端绕过前端同时发起任务，已进入删除流程或已经删除的会话也不得接受新的消息写入。删除失败时，不返回成功，也不从前端历史列表移除会话。

## 测试策略

### 存储层

- SQLite 删除会话、消息、摘要和来源记忆；
- PostgreSQL 实现与 SQLite 保持契约一致；
- `pgvector` 支持按租户、用户和来源会话删除记忆；
- 删除一个会话不会影响其他会话或其他用户的记忆；
- 任一步骤失败时不报告成功。

### API 层

- 当前用户能够删除自己的空会话和有消息会话；
- 其他租户、其他用户和错误 Agent 归属统一返回 `404`；
- `running` 与 `waiting_for_approval` 的父运行或子运行均返回 `409`；
- 已结束运行不阻止删除，并在删除后仍可查询运行追踪；
- 重复删除返回 `404`。

### 前端层

- 会话项结构不存在嵌套按钮；
- 删除按钮悬停、聚焦及移动端可见性正确；
- 确认、取消、`Esc` 和重复提交保护正确；
- 删除当前会话时重置聊天状态；
- 删除非当前会话时保持当前聊天；
- `404`、`409` 和网络错误有明确反馈。

### 回归验证

运行单元测试、集成测试、静态检查和类型检查。浏览器验证如需启动本地服务，测试完成后立即停止本次启动的服务进程。

## 非目标

本次不实现：

- 会话恢复或回收站；
- 批量删除；
- 删除运行追踪、审计事件、运行产物或指标；
- 管理员代用户删除；
- 自动取消正在执行的任务。
