# 小红书动态文字图片卡片设计

## 背景

当前 `xhs_text_image` 发布链路只保存一个 `card_text` 字符串。Playwright 打开小红书“写文字”页面后，只填写第一个编辑器并立即点击“生成图片”。小红书是否分页完全取决于平台内部启发式规则，因此较短正文通常只生成一张图片，框架也无法在审核、审批和发布追踪中表达实际卡片数量。

目标是根据已审核内容动态生成 3–8 张文字图片，封面计入总数，并确保审核内容、审批哈希和浏览器实际填写的卡片完全一致。

## 设计原则

1. 卡片分页必须在发布包冻结前完成，不能在浏览器执行阶段临时让 LLM 改写。
2. 分页只拆分和重排已审核文案，不新增事实、数据或观点。
3. 发布包必须记录每一页内容，内容哈希必须覆盖全部页面及样式。
4. 浏览器实际创建的编辑器数量必须与发布包一致，数量不一致时停止发布。
5. 上传图片模式保持不变；动态卡片只作用于 `xhs_text_image`。

## 内容模型

发布内容中的单一 `card_text` 替换为：

```python
{
    "card_pages": [
        "封面文案",
        "正文页 1",
        "正文页 2",
    ],
    "card_style": "涂鸦",
}
```

`card_pages` 必须是 3–8 个非空字符串。上传图片模式下固定为空列表。`publication_content_hash()` 对规范化后的完整 `card_pages` 计算哈希，因此任何一页内容、顺序或样式变化都会使原审批失效。

## 动态分页算法

新增纯函数负责根据标题和正文规划卡片：

1. 第 1 页是封面，内容为标题和正文中的第一条核心引导句。引导句来自原文，不生成新事实。
2. 正文按段落、换行和中英文句末标点切分为语义片段。
3. 根据正文有效字符数和目标每页字符数计算正文页数量。
4. 总页数限制在租户配置的最小值与最大值之间，默认 3–8 张。
5. 在保持原始片段顺序的前提下，将片段均衡分配到正文页；超长单句允许在字符边界切分，但不丢失字符。
6. 极短正文仍拆成两个正文页以满足最少 3 张；不填充虚构内容。
7. 长正文最多使用 7 个正文页，全部原文仍须保留，不得静默截断。

默认配置：

```json
{
  "text_image_min_pages": 3,
  "text_image_max_pages": 8,
  "text_image_target_chars_per_page": 180
}
```

全局环境配置使用 `AGENTKIT_XHS_TEXT_IMAGE_MIN_PAGES`、`AGENTKIT_XHS_TEXT_IMAGE_MAX_PAGES` 和 `AGENTKIT_XHS_TEXT_IMAGE_TARGET_CHARS_PER_PAGE`；租户 `social_growth` 配置可覆盖全局值。配置必须满足 `3 <= min_pages <= max_pages <= 8`。

## 发布链路

### 发布包准备

`resolve_publish_content()` 在媒体策略确定为 `xhs_text_image` 时执行分页，并将 `card_pages` 和 `card_style` 写入不可变发布包。Mock Provider 和 Playwright Provider 使用同一个解析函数，避免测试与真实发布行为分叉。

### 审批与延迟动作

延迟发布动作传递完整 `card_pages`，不再传递 `card_text`。审批时展示的正文仍是原始文章正文；运行追踪中增加卡片总数和分页内容摘要。由于分页是确定性的派生结果，审核正文与卡片内容保持一致。

### Playwright 执行

1. 打开“写文字”。
2. 等待第一个可编辑卡片并填写封面。
3. 对剩余每一页点击可见且可操作的“再写一张”。
4. 每次点击后等待编辑器数量增加，再填写新增编辑器。
5. 校验所有编辑器的数量和值与 `card_pages` 完全一致。
6. 点击“生成图片”，选择配置样式并进入发布编辑页。
7. 在生成结果可观测时校验生成图片数量；页面未暴露稳定计数节点时，至少保留编辑器阶段的强校验和诊断截图。

“再写一张”按钮采用可见文本、角色和稳定容器组合定位，不使用坐标点击。若按钮缺失、达到平台上限前不可用、编辑器未增加或填写内容未持久化，抛出 `BrowserPageChanged`，并在点击发布前终止。

## 可观测性与错误处理

日志记录以下非敏感信息：

- 计划卡片数；
- 已创建卡片数；
- 每一步耗时；
- 失败阶段和页面诊断路径。

日志不输出完整正文。失败信息区分配置非法、卡片规划失败、创建按钮不可用、编辑器数量不一致和内容未持久化，便于运行追踪定位根因。

## 测试策略

1. 纯函数测试：短正文生成 3 张、中等正文动态增加、长正文限制为 8 张、字符不丢失、封面计入总数。
2. 内容契约测试：`card_pages` 参与哈希；页面顺序或内容变化会改变哈希；上传模式为空列表。
3. Provider 测试：Mock 与 Playwright 准备相同卡片计划。
4. Playwright 适配器测试：逐页创建和填写、数量校验、按钮缺失、编辑器不增长、内容未持久化。
5. 配置测试：默认值、环境变量、租户覆盖和非法范围。
6. 回归测试：上传图片发布、审批哈希、延迟动作和现有 XHS 工作流不受影响。

## 不在本次范围

- 本地渲染小红书风格图片；
- 使用 LLM 为每一页重新创作文案；
- 自动选择不同卡片样式；
- 超过 8 张时拆成多篇笔记；
- 依赖小红书内部自动分页规则。
